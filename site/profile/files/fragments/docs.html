Help

This test environment will ensure that your browser meets minimum requirements and that your network configuration allows you to access all the services we run in the classroom. If the test fails to launch when you click the button, then please inform us of the issue by responding to the email you received or by directly emailing eduteam@puppet.com.

FAQ

  • Where can I find instructions for using the classroom tools?

    We use a tool that allows each student to perform classroom exercises by writing code in a graphical editor and enforcing it on a virtualized agent node. Read the tutorial.

  • Where can I find tutorials on using the Git server?

    We run a self-hosted Git server called Gitea. While it does not have very good end-user documentation, the experience is very close to that of GitHub. We suggest that if you want to practice your Git skills before class that you sign up for a GitHub account and then try out this tutorial.

  • What firewall configuration is needed?

    We run several services that you will need to be able to access via web browser. You may need to ask your network administrator to add rules allowing access.

    • Puppetfactory classroom manager: HTTP on port 80
    • Puppet Enterprise Console: HTTPS on port 443
    • Gitea Repository Server: HTTP on port 3000
    • Courseware Presentation: HTTP and Websocket on port 9090
    • SSH access (optional): SSH on port 22
    • Zoom.us network requirements

  • What are the browser requirements?

    Any modern browser should work. In general, we recommend one of the following.

    • Relatively current versions of Firefox, Chrome, or Safari.
    • Microsoft Edge
    • Internet Explorer version 11+

  • Do I need to install a PDF reader?

    Probably not. Most modern browsers have a PDF reader built-in or use a platform standard reader. If that is not the case for you, then readers such as Adobe Acrobat Reader can be used free of charge.

  • How can I correct issues with the video conferencing (Zoom.us) system?

    Check the Zoom.us network requirements. Their support portal might also have other suggestions.

  • How do I accept a self-signed certificate?

    You can find official instructions in the Puppet Enterprise documentation.

  • What if I can't accept a self-signed certificate?

    Please email eduteam@puppet.com and let us know. We may be able to help you.

  • What can I do to mitigate a slow network?

    If you have limited bandwidth accessible to you, then you can try some techniques to make the most of what you have.

    • Are there other locations you can work from? Perhaps your office has more bandwidth. Or perhaps home does!
    • Try shutting down competing services. For example, if you normally stream music with Spotify or Pandora, you might try turning it off and only playing music locally if you need it.
    • Try turning the camera off so that you're not streaming live video back to the instructor. It's always good to see who you're talking with, but it's better for the classroom to work!
    • Try rebooting your router/modem. Sometimes they work better after a fresh start.

  • The classroom web terminal logged me out!

    In the quest for power efficiency and longer battery life, many browsers will suspend web pages when you've not actively working with them. This means that if the terminal is in a window or tab that you're not actively viewing, your browser might stop maintaining the connection.

    The Puppetfactory classroom manager will attempt to restore the connection when you come back to the page, but if it's been a very long time it may not be able to do so. While logging back in is a minor annoyance, you will not lose any work. If you have to do so more than once or twice a day, please let your instructor know.
